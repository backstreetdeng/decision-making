### 3. 提出架构重设计方案

#### 问题分析
```
❌ 当前错误架构：
用户 → workflow.py (Python流程控制) → SSE (流程控制) → 固定输出

问题：
1. workflow.py 是固定流程，AI无法动态决策
2. SSE 负责流式展示同时控制流程
3. Skills 调用被 Python wrapper 固定
```

```
✅ 正确架构（基于 agents-orchestrator）：
用户 → OpenProse Workflow (AI编排层) → SSE (仅流式展示)
                                    ↓
                              Skills (OpenClaw系统)
```

#### 架构原则
1. **AI 动态编排 > 固定流程控制**
2. **SSE 只做展示，不控制流程**
3. **Skills 通过 OpenClaw 系统调用**
4. **记忆持久化是基础**

### 5. OpenClaw Subagent vs Multi-Agent 研究

**关键发现**：通过研究官方文档，明确了两种架构：

| 模式 | 性质 | 生命周期 | 适用场景 |
|------|------|----------|-----------|
| **Multi-Agent** | 持久独立Agent | 永久运行 | 长期角色分工、多入口路由 |
| **Sub-agent** | 临时任务 | 任务完成即销毁 | 一次性任务、并行执行 |

**参考文章**：
- 阿里云开发者社区文章：Multi-Agent vs 主Agent+Sub-Agent 完整指南
- GitCode教程：OpenClaw多Agent协作创建
